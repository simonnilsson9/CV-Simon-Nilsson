Web Content Accessibility Guidelines

Guidelines

  • Tillhandahålla textalternativ för icke-textinnehåll (WCAG 1.1): Se till att allt icke-textinnehåll som bilder, grafik eller ljudfiler har alternativa textbeskrivningar för att hjälpa personer med synnedsättning.
    - Detta görs genom att jag har satt alternativ text till varje bild.
  • Användbart och navigerbart innehåll (WCAG 2.4): Se till att webbsidan är användbar och lätt att navigera för personer med olika funktionshinder. Detta inkluderar att tillhandahålla logiska navigationsstrukturer och interaktionsmönster.
    - Detta görs genom att jag har en tydlig navigationsbar och tydliga strukturer på hemsidan.
  • Skalbar och anpassningsbar innehåll (WCAG 1.4): Webbsidor bör kunna skalas upp utan att förlora funktionalitet eller förstöra layouten, för att möta behoven hos användare som använder förstoring eller andra anpassningsverktyg.
    - Detta görs genom att jag har gjort sidan responsiv vilket gör att den anpassar sig utifrån vilken skärmstorlek som används.
  • Användning av ARIA-länder (WCAG 4.1.2): Använd WAI-ARIA (Accessible Rich Internet Applications) för att förbättra tillgängligheten för dynamiskt innehåll, särskilt för användare av skärmläsare.
    - Detta görs genom att jag har gjort sidan responsiv vilket gör att den anpassar sig utifrån vilken skärmstorlek som används.
  • Tillgänglighet för färgkontrast (WCAG 1.4.3): Se till att det finns tillräcklig kontrast mellan text och bakgrund för att underlätta läsbarheten för personer med nedsatt syn.
    - Detta uppfylls eftersom jag ej har tagit allt för skarpa färger utan hållt mig till svart och vit vilket är lätt att urskilja.
  • Tillgänglighet vid inmatning (WCAG 2.1): Se till att användare kan navigera och interagera med webbsidan med olika inmatningsenheter som tangentbord eller pekenheter.
    -Man kan använda muspekaren för att navigera. Vet ej riktigt hur man implementerar så den är helt anpassad för tangentbord.
  • trukturerad semantik (WCAG 4.1.1): Använd lämplig HTML-struktur för att markera upp innehållet på ett semantiskt korrekt sätt, vilket underlättar förståelsen och navigeringen för användare och hjälper till med skärmläsning.
    - Jag använder mig av tydliga klasser och på så sätt är det väldigt tydligt att följa min struktur, både i HTML och CSS.
  • Tillgänglighet för rörligt innehåll (WCAG 2.2): Se till att rörligt innehåll, såsom blinkande eller rullande text, kan stoppas, pausas eller döljas för att undvika distraherande eller potentiellt farliga effekter för användare med kognitiva funktionshinder.
    - Har inget rörligt innehåll.
  • Tillgänglighet för ljudkontroll (WCAG 2.5): Se till att användare har möjlighet att kontrollera ljudinnehåll, inklusive möjligheten att pausa, spela upp och justera ljudnivån, för att möta behoven hos användare med hörselskador.
    Har inget rörligt innehåll men hade varit lätt att fixa utifall att jag hade haft det.
  • Tillgänglighet för innehållsuppdateringar (WCAG 3.2): Se till att användare kan upptäcka och förstå när innehållet på webbsidan har uppdaterats eller förändrats, särskilt om förändringarna sker dynamiskt eller utan användarinteraktion.
  • Tillgänglighet för anpassningsbara gränssnitt (WCAG 4.1.4): Möjliggör anpassning av gränssnittet genom att tillhandahålla alternativa layouter, färger och formatinställningar för att möta individuella användares preferenser och behov.
    - Inte implementerat men man hade kunant fixa ett alternativt tema.
